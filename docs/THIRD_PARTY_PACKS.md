# Third-Party Packs

JoyBoy supports optional local packs. Packs can add routing rules, prompt assets, model sources, UI overrides, or experimental workflows without modifying the public core.

Third-party packs are external addons. They are not part of JoyBoy's public core, are not bundled with the public repository, and are not official JoyBoy releases. Review any pack before installing it, and only use packs that comply with your local laws, model licenses, and hosting platform policies.

## Installing a pack

1. Download the pack zip or folder.
2. Open JoyBoy.
3. Open `Addons` from the sidebar.
4. Open the local packs section.
5. Import the zip or local folder.
6. Activate the pack.

JoyBoy expects a pack to contain a `pack.json` manifest and an `assets/` folder.

## Specialized external addons

Some external addons may extend JoyBoy toward narrower or more sensitive workflows, including broader human-image pipelines than the default public core exposes. These addons remain separate from the public JoyBoy project.

Rules of thumb:

- do not use packs for non-consensual intimate imagery;
- do not use packs for minors or ambiguous-age subjects;
- do not publish explicit preview media in the public JoyBoy docs or README;
- keep specialized packs opt-in and separate from the public core;
- follow the model license and the hosting platform rules.
