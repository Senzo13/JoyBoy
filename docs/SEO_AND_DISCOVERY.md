Use this checklist to keep JoyBoy easy to find on GitHub without keyword spam.

Recommended repository description:

```text
Local-first AI workstation and harness: private ChatGPT/Grok-style chat, SDXL image editing, video experiments, Ollama, model imports, addons, and Codex-style project mode.
```

Recommended website/homepage:

```text
https://github.com/Senzo13/JoyBoy
```

## README Discovery Terms

Keep these phrases naturally present in the README and docs:

- local AI workstation
- local AI harness
- LLM harness
- agent harness
- model orchestration harness
- open source ChatGPT alternative
- Grok-like local assistant
- offline AI assistant
- Ollama chat
- Stable Diffusion / SDXL image generation
- image editing and inpainting
- text-to-image generation
- local image-to-video
- Hugging Face and CivitAI model imports
- local addons / packs
- 8 GB VRAM local AI
- Codex-style project mode

Do not hide keywords in comments or invisible text. GitHub discovery should stay readable and useful.

## Social Preview

Set a GitHub social preview image once a real screenshot exists.

Recommended image:

- 1280x640 PNG
- dark UI screenshot or branded hero
- visible JoyBoy logo
- one short line: "Local AI workstation"
- no explicit or sensitive content

## Release SEO

Use release titles that describe what changed:

- `JoyBoy v0.2 - Local Addons and Model Imports`
- `JoyBoy v0.3 - Project Mode Preview`
- `JoyBoy v0.4 - Faster 8GB VRAM Image Workflows`

Release notes should include keywords naturally through actual features, not tags pasted at the bottom.

## Public Safety

Keep the public core neutral. Optional third-party packs may target mature workflows where legal, consensual, and compliant with platform policies, but those packs should remain outside the public core.
