# Roadmap

## Near term

- keep onboarding reliable on Windows, macOS, and Linux
- strengthen Doctor coverage and actionable fixes
- simplify provider/model import UX
- keep the public core easy to understand for contributors

## Public core work

- continue reducing duplicated routing logic
- make feature exposure maps more explicit
- improve install diagnostics and packaging
- document architecture and extension boundaries

## Local extension work

- improve local pack validation
- support more optional pack kinds
- document compatibility expectations for pack authors

## Good first contributions

- UI polish that does not change architecture
- docs fixes and install clarifications
- Doctor checks with clear remediation text
- provider/model import improvements
- tests for config, packs, and onboarding flows

## Help wanted

- packaging and reproducible setup
- smoke tests for launcher flows
- public mirror review tooling
- docs examples and contributor experience
