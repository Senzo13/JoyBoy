JoyBoy en bref
Un harness IA local
JoyBoy réunit chat LLM, recherche web, génération d’images, inpainting intelligent et vidéo dans un seul cockpit local. Il adapte ses modèles et réglages à ta machine.
Flux principaux
Chat, web et terminal
Utilise le modèle Ollama choisi pour discuter, raisonner, chercher sur le web si nécessaire et travailler avec le contexte local.
Image et inpainting
Décris ton intention en langage naturel. Le Smart Router choisit le masque, la force, les modèles de segmentation, ControlNet Depth et les références visage/style quand elles sont utiles.
Vidéo
Génère une vidéo depuis du texte ou anime une image avec un prompt. Les presets s’adaptent au modèle vidéo et à la VRAM disponible.
Configuration et extensions
Doctor et accès modèles
Va dans Modèles pour vérifier Python/GPU/Ollama, brancher Hugging Face ou CivitAI, importer des modèles et voir les actions à corriger.
Packs locaux
Le core public reste neutre. Les packs locaux ajoutent des règles de routing, prompts, sources de modèles ou surfaces UI sur ta machine sans les embarquer dans le repo public.
Performance
Si un résultat est lent ou instable, commence par le Doctor, puis ajuste le modèle, les steps, la qualité vidéo et le mode de génération. Les gros modèles demandent plus de VRAM et d’espace disque.
Créer un addon
Un addon JoyBoy est un pack local importable
Un addon peut ajouter des règles de routing, des prompts, des sources de modèles ou des surfaces UI sans modifier le core public.
Contrat
Chaque addon expose un pack.json versionné, validé avant import et copié dans le dossier local des packs.
Assets
Les fichiers assets peuvent rester vides au départ, puis alimenter le routeur, les prompts, les modèles ou l’UI.
Import
Zippe le dossier de l’addon ou colle son chemin local dans Paramètres > Modèles > Packs locaux.
Structure minimale
my-addon/
  pack.json
  assets/
    router_rules.json
    prompt_assets.json
    model_sources.json
    ui_overrides.json
Des templates vides et un exemple neutre sont fournis dans docs/addons/.
